### 문제 설명

<p>신종 바이러스인 웜 바이러스는 네트워크를 통해 전파된다. 한 컴퓨터가 웜 바이러스에 걸리면 그 컴퓨터와 네트워크 상에서 연결되어 있는 모든 컴퓨터는 웜 바이러스에 걸리게 된다.</p>

<p>예를 들어 7대의 컴퓨터가 <그림 1>과 같이 네트워크 상에서 연결되어 있다고 하자. 1번 컴퓨터가 웜 바이러스에 걸리면 웜 바이러스는 2번과 5번 컴퓨터를 거쳐 3번과 6번 컴퓨터까지 전파되어 2, 3, 5, 6 네 대의 컴퓨터는 웜 바이러스에 걸리게 된다. 하지만 4번과 7번 컴퓨터는 1번 컴퓨터와 네트워크상에서 연결되어 있지 않기 때문에 영향을 받지 않는다.</p>

### 입력 

 <p>첫째 줄에는 컴퓨터의 수가 주어진다. 컴퓨터의 수는 100 이하인 양의 정수이고 각 컴퓨터에는 1번 부터 차례대로 번호가 매겨진다. 둘째 줄에는 네트워크 상에서 직접 연결되어 있는 컴퓨터 쌍의 수가 주어진다. 이어서 그 수만큼 한 줄에 한 쌍씩 네트워크 상에서 직접 연결되어 있는 컴퓨터의 번호 쌍이 주어진다.</p>

### 출력 

 <p>1번 컴퓨터가 웜 바이러스에 걸렸을 때, 1번 컴퓨터를 통해 웜 바이러스에 걸리게 되는 컴퓨터의 수를 첫째 줄에 출력한다.</p>

### Idea

#### 처음 생각

항상 이와 같은 문제 -재귀를 돌면서 값을 하나씩 더한다던지 하는- 가 등장하면, 재귀 함수를 __"반환값이 있는 함수"__ 로 먼저 고안하게 된다.  
그런데 경험상, 결국에는 반환값 없는 `void` 함수로 수정하게 되었다...  
아무래도 전역변수를 두어 실시간으로 값을 변경하는 방식이 직관적이라 그런 것 같은데, 반환값이 있는 재귀 함수로도 얼마든지 구현할 수 있다.  

``` text
<ans 값을 반환하는 재귀 함수>
- 재귀 함수 내부에서 쓸 지역 변수를 하나 선언한다. (int ans = 1;)
- 연결된 노드 중 방문하지 않은 것을 골라 재귀를 돈다. (위와 같다)
- ⭐ 재귀를 돌아 반환되는 `ans` 값을 현재의 내 `ans` 에 더한다 !!  ( count += findVirus(next); )
- 취합된 ans 를 return 한다.
```

``` cpp
int findVirus(int root) {
    visited[root] = true;
    int ans = 1; // 나 자신도 감염

    for (int next : V[root]) {
        if (!visited[next]) {
            // !재귀를 돌며 next가 찾아온 바이러스 수를 내 결과에 더한다!
            ans += findVirus(next); 
        }
    }
    return ans; // 나와 나에게 연결된 모든 바이러스 수를 전달
}
```
전체 코드는 위와 같다.   
나에게서 파생되어 재귀를 돌며 열심히 모은 값을 정작 "취합하지 않아서" 의도치 않은 값이 계속 나왔던 것 같다.
