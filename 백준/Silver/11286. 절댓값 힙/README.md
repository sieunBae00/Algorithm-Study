### 문제 설명

<p>절댓값 힙은 다음과 같은 연산을 지원하는 자료구조이다.</p>

<ol>
	<li>배열에 정수 x (x ≠ 0)를 넣는다.</li>
	<li>배열에서 절댓값이 가장 작은 값을 출력하고, 그 값을 배열에서 제거한다. 절댓값이 가장 작은 값이 여러개일 때는, 가장 작은 수를 출력하고, 그 값을 배열에서 제거한다.</li>
</ol>

<p>프로그램은 처음에 비어있는 배열에서 시작하게 된다.</p>

### 입력 

 <p>첫째 줄에 연산의 개수 N(1≤N≤100,000)이 주어진다. 다음 N개의 줄에는 연산에 대한 정보를 나타내는 정수 x가 주어진다. 만약 x가 0이 아니라면 배열에 x라는 값을 넣는(추가하는) 연산이고, x가 0이라면 배열에서 절댓값이 가장 작은 값을 출력하고 그 값을 배열에서 제거하는 경우이다. 입력되는 정수는 -2<sup>31</sup>보다 크고, 2<sup>31</sup>보다 작다.</p>

### 출력 

 <p>입력에서 0이 주어진 회수만큼 답을 출력한다. 만약 배열이 비어 있는 경우인데 절댓값이 가장 작은 값을 출력하라고 한 경우에는 0을 출력하면 된다.</p>

### Idea

#### 처음 생각

`pair<int, bool>` 을 담는 `priority_queue` 를 만들자.  
`bool` 에는 절댓값 처리(`abs()`) 를 했는지 여부를 저장하자.
</br>

- 🛑 구현의 어려움  
`priority_queue` 에서 맨 위 두 개를 뽑아 둘을 비교하려니 큐에 원소가 하나밖에 없는 경우 에러가 난다...  
그 외에도 비교/출력 부분에서 고려해야 할 것들이 많아 코드가 복잡해진다.  
--> 더 나은 방법이 있을 것 같은데...
</br>

> 💡 `pair` 을 정렬할 때, `first` 값이 같으면 `second` 값을 비교한다는 사실을 이용한다.  
> * 사실 문제에서도 __'출력은 원래의 값'__ 이므로, 원래의 값을 함께 저장하라는 힌트가 되기도 한다.  

-->  
`pair<int, int>` 을 담는 `priority_queue` 를 만들자.  
`frist` 에는 절댓값 처리(`abs()`) 한 값을, __`second` 에는 '원래 값'__ 을 저장하자.  
애초에 `priority_queue` 에 저장될 때 `second` 까지 고려해 정렬해 준다. 절댓값이 같아도, 자동으로 원래 값이 작은 쪽이 `top` 이 된다.
