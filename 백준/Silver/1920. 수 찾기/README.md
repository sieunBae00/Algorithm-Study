### 문제 설명

<p>N개의 정수 A[1], A[2], …, A[N]이 주어져 있을 때, 이 안에 X라는 정수가 존재하는지 알아내는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 자연수 N(1 ≤ N ≤ 100,000)이 주어진다. 다음 줄에는 N개의 정수 A[1], A[2], …, A[N]이 주어진다. 다음 줄에는 M(1 ≤ M ≤ 100,000)이 주어진다. 다음 줄에는 M개의 수들이 주어지는데, 이 수들이 A안에 존재하는지 알아내면 된다. 모든 정수의 범위는 -2<sup>31</sup> 보다 크거나 같고 2<sup>31</sup>보다 작다.</p>

### 출력 

 <p>M개의 줄에 답을 출력한다. 존재하면 1을, 존재하지 않으면 0을 출력한다.</p>

### Idea  

- 탐색 --> 이진 탐색 트리를 이용한다.  
<br>

물론 `vector` -> `sort()` -> `binary_search()` 를 이용할 수도 있지만 (이 문제에서는 그쪽이 더 낫다고도 할 수 있겠지만)  
이번에는 `BST` 처럼 사용 가능한 `set` 을 활용해 보았다.  


#### `set` 의 사용 익히기  

`set` 은 내부적으로 균형잡힌 `red-black tree` 로 구현되어 있으며, 실제로 데이터를 `node` 에 담는다.  
`node` 에는 `데이터 값`과 `왼쪽 자식 주소`, `오른쪽 자식 주소`, `부모 주소` 가 포함되어 저장된다.  
<br>

삽입/삭제 시 `.insert()` , `.erase()` 를 사용한다. 삽입 시 자동으로 균형잡힌 트리를 형성하며 정렬된다.  
삽입하려는 데이터가 이미 존재한다면 삽입 연산을 무시한다.  
(`set` 은 __중복 허용 안함__ - 이진 탐색 트리의 성질)  
<br>

탐색 시 `.find()` 를 사용한다. 원하는 데이터를 찾으면 그 데이터의 위치를 반환한다.  
만약 데이터를 찾지 못하면 `(마지막 데이터+1)칸` 에 도달해 그 칸의 주소를 반환한다. 이는 `.end()` 와 같은 값이다.  
<br>

데이터의 숫자를 세기 위해서는 `.count()` 를 사용한다.  
다만 `set` 은 중복을 허용하지 않으므로 `.count()` 의 값은 반드시 `0` 또는 `1` 이 된다.  

---

#### `set` v.s. `vector`  

- `set` 은 내부적으로 트리로 구현된다. 반면 `vector` 는 일렬로 늘어선 배열이다.  
<br> 

둘 다 `find` 연산의 속도는 비슷하지만, 삽입/삭제 시 `vector` 는 `shift` 연산 (`O(n)`) 을 해야 한다.  
따라서 삽입/삭제가 빈번한 이진 탐색 상황에서는 `set` 을 사용해야 한다.
