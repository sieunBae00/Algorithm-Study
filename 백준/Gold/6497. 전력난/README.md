### 문제 설명

<p>성진이는 한 도시의 시장인데 거지라서 전력난에 끙끙댄다. 그래서 모든 길마다 원래 켜져 있던 가로등 중 일부를 소등하기로 하였다. 길의 가로등을 켜 두면 하루에 길의 미터 수만큼 돈이 들어가는데, 일부를 소등하여 그만큼의 돈을 절약할 수 있다.</p>

<p>그러나 만약 어떤 두 집을 왕래할 때, 불이 켜져 있지 않은 길을 반드시 지나야 한다면 위험하다. 그래서 도시에 있는 모든 두 집 쌍에 대해, 불이 켜진 길만으로 서로를 왕래할 수 있어야 한다.</p>

<p>위 조건을 지키면서 절약할 수 있는 최대 액수를 구하시오.</p>

### 입력 

 <p>입력은 여러 개의 테스트 케이스로 구분되어 있다.</p>

<p>각 테스트 케이스의 첫째 줄에는 집의 수 m과 길의 수 n이 주어진다. (<em>1 ≤ m ≤ 200000</em>, <em>m-1 ≤ n ≤ 200000</em>)</p>

<p>이어서 n개의 줄에 각 길에 대한 정보 x, y, z가 주어지는데, 이는 x번 집과 y번 집 사이에 양방향 도로가 있으며 그 거리가 z미터라는 뜻이다. (<em>0 ≤ x, y < m</em>, <em>x ≠ y</em>)</p>

<p>도시는 항상 연결 그래프의 형태이고(즉, 어떤 두 집을 골라도 서로 왕래할 수 있는 경로가 있다), 도시상의 모든 길의 거리 합은 2<sup>31</sup>미터보다 작다.</p>

<p>입력의 끝에서는 첫 줄에 0이 2개 주어진다.</p>

### 출력 

 <p>각 테스트 케이스마다 한 줄에 걸쳐 절약할 수 있는 최대 비용을 출력한다.</p>

### Idea

#### 처음 생각

`N`개의 정점을 모두 연결하는 최소 간선 수는 `N-1` 이다. 현재의 간선 수(`m`) 에서 간선 수가 `N-1` 개가 될 때까지 하나씩 간선을 제거해 나가면 된다.  
제거할 때에는 도로의 길이가 긴 것부터(greedy) 제거한다.  
  
**단, 어떤 집이 고립(그 집으로 향하는 도로가 하나도 없는 경우) 되지 않도록 해야 하므로,  
어떤 노드에 연결된 간선이 그 하나 뿐이라면, 그것은 제거하지 않고 다음 간선으로 넘어간다.  
  
##### 구현

도로의 길이가 긴 것부터 제거 -> 도로의 길이와, 그 도로가 연결하는 집(`{z, x, y}`) 정보를 저장하는  `priority_queue` 를 둔다.  
도로의 길이를 기준으로 내림차순 정렬해야 하므로 tuple의 첫 번째 요소를 `z`(도로의 길이) 로 둔다.  
  
도로의 길이와 그에 연결된 집이 pq 에 저장되므로, 각 집에 연결된 도로가 몇 개인지를 저장하는  
'`graph` 2차원 배열'에는 도로의 길이를 따로 저장하지 않아도 된다.

#### 올바른 방법

> 💡 [생각 바꾸기] 긴 도로부터 제거 (×) --> __짧은 도로부터 연결 (o)__ !!   
> 간선 수가 m-1 이 될 때까지 반복하고, 중간에 'cycle' 이 발생하게 될 경우 그것은 불필요한 간선이므로 연결하지 않고 다음으로 넘어간다.  
> *cycle 확인 방법? --> __Union-Find (유니온 파인드)__ !!  
  
`Union()`, `find()` 함수에서 보스를 찾고, 보스가 다르다면 병합까지 완료하도록 구현하였다.  
(`main()` 에서는 그 외 도로 개수, 도로의 길이 총합 계산 등을 담당한다)  
</br>
  
- ⚠️ 런타임 에러 - `WithoutReturning`  
  다음과 같이 find() 함수를 작성해 제출하였더니 WithoutReturning 에러가 발생했다.  
  ``` cpp
  int find(int curr) {
    if (house[curr] == curr) return curr;
    else find(house[curr]);
  }
  ```
    
  당연한 결과다. 보스를 찾으러 위로 올라가 놓고는, 정작 그 보스의 번호를 `return` 해 주지 않았기 때문이다.  
  (재귀 함수에서, 재귀를 돌 때 그 찾은 값을 넘겨주어야 한다는 것을 잊지 말자..)  
  위 코드에서 else문 부분을  
  ``` cpp
  else return find(house[curr]);
  ```
  로 바꾸어 주면 해결된다.!
