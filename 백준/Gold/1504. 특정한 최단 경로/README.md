### 문제 설명

<p>방향성이 없는 그래프가 주어진다. 세준이는 1번 정점에서 N번 정점으로 최단 거리로 이동하려고 한다. 또한 세준이는 두 가지 조건을 만족하면서 이동하는 특정한 최단 경로를 구하고 싶은데, 그것은 바로 임의로 주어진 두 정점은 반드시 통과해야 한다는 것이다.</p>

<p>세준이는 한번 이동했던 정점은 물론, 한번 이동했던 간선도 다시 이동할 수 있다. 하지만 반드시 최단 경로로 이동해야 한다는 사실에 주의하라. 1번 정점에서 N번 정점으로 이동할 때, 주어진 두 정점을 반드시 거치면서 최단 경로로 이동하는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 정점의 개수 N과 간선의 개수 E가 주어진다. (2 ≤ N ≤ 800, 0 ≤ E ≤ 200,000) 둘째 줄부터 E개의 줄에 걸쳐서 세 개의 정수 a, b, c가 주어지는데, a번 정점에서 b번 정점까지 양방향 길이 존재하며, 그 거리가 c라는 뜻이다. (1 ≤ c ≤ 1,000) 다음 줄에는 반드시 거쳐야 하는 두 개의 서로 다른 정점 번호 v<sub>1</sub>과 v<sub>2</sub>가 주어진다. (v<sub>1</sub> ≠ v<sub>2</sub>, v<sub>1</sub> ≠ N, v<sub>2</sub> ≠ 1) 임의의 두 정점 u와 v사이에는 간선이 최대 1개 존재한다.</p>

### 출력 

 <p>첫째 줄에 두 개의 정점을 지나는 최단 경로의 길이를 출력한다. 그러한 경로가 없을 때에는 -1을 출력한다.</p>

### Idea

다익스트라 알고리즘 자체에 특정 정점(`v1`, `v2`) 를 거치는지 확인하는 로직을 추가하기는 어렵다.  
따라서 경로를 분할해서 최단 경로를 결정한다.
</br>

💡 정점 A에서 출발하여 정점 B에 이르는 경로 중, 정점 v를 거치는 최단 경로는  
`(A -> v 까지의 최단 경로) + (v -> B 까지의 최단 경로)` 임을 이용한다.  
  
정점1 에서 정점N 까지 이동하는데 `v1`, `v2` 를 반드시 거쳐야 한다면, 가능한 경로는 단 두 가지 이다.  
``` text 
경로A) 1 -> v1 -> v2 -> N
경로B) 1 -> v2 -> v1 -> N
```
> 💡 이제 다익스트라를 __여러 번__ 수행하며 각 정점들 간의 거리를 구한 후, 그것들을 더하면  
> 경로 A, B 의 거리를 알아낼 수 있다. 경로 A, B 중 더 작은 쪽이 최단 경로가 된다.  
</br>

- ⚠️ 정점들 간의 거리(`dist[]`) 를 `INT_MAX` 로 초기화하면 안 된다.
  INT_MAX는 C++의 int가 가질 수 있는 가장 큰 값(약 21억) 이다.  
``` cpp
if (dist[next] > dist[nn] + val) {}
```
만약 dijkstra() 함수 내부의 위와 같은 코드에서 dist[nn] 이 INT_MAX 라면, 그 상태에서 val 을 더할 경우  
오버플로우가 발생해 값이 음수가 되어 버린다.  ->  음수인 거리가 큐에 들어가고  ->  거리 자체가 이상해짐.  
</br>

- ⚠️ 예외를 생각하기: `v1`이 `1`이고, `v2`가 `N`일 경우?  
``` cpp
if (ans >= INF) cout << -1 << '\n';
```
위 코드에서 주의할 점:  
`v1`이 `1`이고, `v2`가 `N`일 경우가 발생할 수 있다. 이때 `1`에서 `N` 으로의 __경로가 없다면,__ `ans` 는 정확히 `INF` 가 된다. (`0 + INF + 0`)   
따라서 __등호를 붙여 주어야__ 위와 같은 특수한 상황에서도 의도한 결과를 만들어낼 수 있다.
