### 문제 설명

<p>세 개의 장대가 있고 첫 번째 장대에는 반경이 서로 다른 n개의 원판이 쌓여 있다. 각 원판은 반경이 큰 순서대로 쌓여있다. 이제 수도승들이 다음 규칙에 따라 첫 번째 장대에서 세 번째 장대로 옮기려 한다.</p>

<ol>
	<li>한 번에 한 개의 원판만을 다른 탑으로 옮길 수 있다.</li>
	<li>쌓아 놓은 원판은 항상 위의 것이 아래의 것보다 작아야 한다.</li>
</ol>

<p>이 작업을 수행하는데 필요한 이동 순서를 출력하는 프로그램을 작성하라. 단, 이동 횟수는 최소가 되어야 한다.</p>

<p>아래 그림은 원판이 5개인 경우의 예시이다.</p>

<p style="text-align: center;"><img alt="" src="https://onlinejudgeimages.s3-ap-northeast-1.amazonaws.com/problem/11729/hanoi.png" style="height:200px; width:1050px"></p>

### 입력 

 <p>첫째 줄에 첫 번째 장대에 쌓인 원판의 개수 N (1 ≤ N ≤ 20)이 주어진다.</p>

### 출력 

 <p>첫째 줄에 옮긴 횟수 K를 출력한다.</p>

<p>두 번째 줄부터 수행 과정을 출력한다. 두 번째 줄부터 K개의 줄에 걸쳐 두 정수 A B를 빈칸을 사이에 두고 출력하는데, 이는 A번째 탑의 가장 위에 있는 원판을 B번째 탑의 가장 위로 옮긴다는 뜻이다.</p>

### Idea

#### 처음 생각

하노이 탑의 원판의 최소 이동 횟수는 `2^n-1` 이다.  
그리고 하노이 탑은 재귀와 밀접한 관련이 있다.  
  
직접 원판을 이동시키는 방향으로 구현은 불가능해 보이니, 재귀밖에 방법이 없을 것 같은데, 어떻게 구현할 지는 떠오르지 않았다..  
</br>

> '재귀와 왜 연관이 되는가'를 생각해 보자.  
> 💡 맨 밑에 깔린 가장 큰 원판 / 나머지 원판 으로 나누어 생각한다.  
> 맨 밑의 원판을 `출발지->도착지` 로 옯기기 위해서는, 우선 __나머지 원판들이 `출발지->보조기둥` 으로 옯겨져 있어야__ 한다. !!  
  
``` text
1. 맨 밑 원판을 제외한 나머지 원판들을 '출발지->보조기둥' 으로 옮긴다.
2. 맨 밑 원판을 '출발지->도착지' 로 옮긴다.
3. 다시 나머지 원판들을 '보조기둥->도착지' 로 옮긴다.
(원판이 2개 이상인 경우에 대하여, 똑같이 반복 적용한다)
```
</br>

- ⭐ 재귀 함수의 파라미터  
  재귀 함수(`hanoi()`) 파라미터로 __`(원판 개수, 출발지, 도착지, 보조기둥)`__ 을 설정했다.  
  재귀를 돌 때마다 원판 수가 하나씩 작아질 것이고, 출발/도착/보조기둥 이 계속해서 변하기 때문이다. !!  
</br>

- ⚠️ 재귀 함수의 구조  
  항상 재귀 함수에서 실수가 나온다..  
  처음 구현할 때 위의 3단계 로직을 `main()` 함수 안에 넣었다. 그러면? 당연하게도, 재귀가 원하는 방향으로 끝까지 구현되지 않는다.  
  --> 위의 3단계 로직을 `main()` 이 아닌, 재귀 함수(`hanoi()`) 내부로 넣어야 한다. !!
